Die Vision

Ein AI-Unternehmen, in dem 10 Agenten sich in Echtzeit unterhalten — wie eine echte C-Suite. Jeder Agent besitzt die höchste Fachkompetenz seines Bereichs auf dem Weltmarkt. Du siehst jede Konversation live in einem "Boardroom" Dashboard — parallel, transparent, nachvollziehbar.

Vorbilder: Die besten Unternehmen der Welt

Bridgewater Associates

Radikale Transparenz

Ray Dalio's Prinzip: Alle Meetings aufgezeichnet. Ideen werden nach Glaubwürdigkeit gewichtet, nicht nach Hierarchie. "Idea Meritocracy" — das beste Argument gewinnt.

→ Unsere Agenten: Jede Kommunikation wird geloggt und ist einsehbar. Entscheidungen basieren auf Datenqualität, nicht auf Agent-Rang.
Renaissance Technologies

Quantitativer Ansatz

Jim Simons' Medallion Fund: Flache Struktur, geteilte Incentives. Hire smart people from outside finance. Alle Entscheidungen modellbasiert — kein "Bauchgefühl".

→ Unsere Agenten: Jeder Agent hat Zugang zu denselben Daten. Entscheidungen sind reproduzierbar und auditierbar.
Amazon

Two-Pizza Teams + Written Culture

Jeff Bezos: Kleine, autonome Teams. Keine PowerPoints — 6-Seiten-Memos. "Disagree and Commit". Day-1-Mentalität. Single-Threaded Ownership.

→ Unsere Agenten: Jeder Agent "owns" seinen Bereich vollständig. Kommunikation über strukturierte Memos, nicht Chat.
Berkshire Hathaway

Delegierte Autonomie

Warren Buffett: 24 Personen managen $1,16 Billionen. Vertrauen + kulturelle Ausrichtung statt Prozesse. Extreme Dezentralisierung mit klaren Leitplanken.

→ Unsere Agenten: Maximal autonom innerhalb klarer Grenzen. Guardian als "Board of Directors" mit Veto-Recht.

Kritischer Research-Befund

"Slack für AI-Agenten" existiert nicht. Kein Framework bietet echte Inter-Agent-Kommunikation mit Echtzeit-Visualisierung. LangGraph kommt am nächsten (Shared State + Parallel Execution), aber selbst dort "reden" Agenten nicht miteinander — sie lesen/schreiben in einen gemeinsamen State.

Das bedeutet: Wir bauen etwas, das es noch nicht gibt. Das ist gleichzeitig Risiko und Chance — weil es zum Produkt werden kann.

Roadmap: 14 Meilensteine zum Agent Boardroom

Von Foundation bis Full Autonomous C-Suite · Geschätzte Gesamtdauer: 10-14 Wochen

Woche 1 · Voraussetzung für alles
  • Gewerbeanmeldung Magdeburg (Einzelunternehmen, ~€30)
  • Impressum + Datenschutzerklärung (DSGVO-konform)
  • Disclaimer-Templates für alle Content-Typen
  • Affiliate-Werbekennzeichnung nach TMG §6
  • Counsel-Agent: Knowledge Base mit BaFin, MiCA, GlüStV, DAC8, DSGVO laden
GewerbeanmeldungDSGVOMiCA

Phase 2: Infrastructure

Woche 1-2 · Server + Database + Repo
  • Hetzner VPS CX22 bestellen + Docker Setup
  • Supabase Projekt: agents_messages, agents_state, agents_logs Tabellen
  • Supabase Realtime aktivieren (WebSocket für Dashboard)
  • Redis auf VPS installieren (internes Pub/Sub)
  • GitHub Actions CI/CD Pipeline
  • Domain registrieren + Cloudflare DNS
DockerSupabaseRedisGitHub ActionsCloudflare

Phase 3: Message Bus ("Der Konferenzraum")

Woche 2-3 · KERN-INNOVATION
  • Agent Message Protocol definieren (JSON Schema)
  • Supabase Realtime Channels: #strategy, #market-intel, #content, #legal, #trading, #ops, #emergency
  • Python SDK: AgentMessage Klasse (send, subscribe, broadcast)
  • Message Routing: @mention für Direct, #channel für Broadcast
  • Priority System: normal / high / emergency (Guardian-Alerts)
  • Message Persistence in Supabase (Audit Trail)
Supabase RealtimeRedis Pub/SubPython SDKJSON Schema

Phase 4: Agent Base Class

Woche 3 · Das Framework für alle Agenten
  • BaseAgent Klasse: lifecycle (init → think → act → report → sleep)
  • LLM-Integration: Claude Sonnet/Haiku via Anthropic SDK
  • Knowledge Injection: Jeder Agent bekommt sein Fach-Wissen als System Prompt
  • Tool-System: Jeder Agent kann externe APIs aufrufen
  • Circuit Breaker: Max-Retries, Timeout, Token-Limits
  • State Management: Agent-State in Supabase (persistent über Restarts)
