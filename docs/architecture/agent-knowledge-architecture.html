Das Kernproblem
Ein AI-Agent ist nur so gut wie sein Wissen + seine Skills + seine Tools. Ohne Weltklasse-Knowledge sind unsere 17 Agenten nur teure Textgeneratoren.
"Garbage In, Garbage Out" — gilt für AI mehr als für jedes andere System. Die Qualität des Inputs definiert die Qualität des Outputs. Kein Prompt Engineering der Welt kompensiert fehlendes Fachwissen.
❌

Aktueller Stand: Schwach

Unsere Agenten nutzen aktuell nur das Basis-Wissen von Claude (Training bis Mai 2025). Kein aktuelles Marktwissen, keine spezialisierten Frameworks, keine DACH-spezifische Regulatorik-Tiefe, keine Platform-Algorithmus-Expertise.

→ Ein ORACLE ohne aktuelle On-Chain-Daten ist blind. Ein PUBLISHER ohne Platform-Algorithmus-Wissen verschwendet Content.

1. Echtzeit-Daten: Marktpreise, On-Chain, Sentiment, Trending Topics
2. Domänen-Expertise: Whitepapers, Frameworks, Best Practices, Regulatorik
3. Platform-Intelligence: Algorithmen, Timing, Format-Specs, Viral Mechanics

→ Weltklasse = Echtzeit × Expertise × Platform-Wissen

Knowledge Architecture — RAG + Tools + Prompts
So bauen wir Weltklasse-Expertise in jeden einzelnen Agenten ein. 3 Schichten, 4 Datentypen, kontinuierliche Aktualisierung.
Gartner 2026: "Über 60% aller Enterprise-AI-Anwendungen werden Agentic Components haben — aber 40% der Projekte scheitern an schlechter Architektur." Wir bauen es richtig.

🧬 Die 3-Schichten-Architektur pro Agent

Schicht 1: Statische Knowledge Base

Was: Curated Documents in Supabase Vector DB (pgvector). Whitepapers, Regulatorik-Texte, Frameworks, Best Practices, Steuergesetze, Platform-Guides.

Wie: Dokumente werden gechunkt (512-1024 Tokens), embeddet (OpenAI ada-002 oder Anthropic Embeddings), und in Supabase gespeichert. Bei jedem Task: Semantic Search → Top 5 relevante Chunks → Context Injection.

Update-Zyklus: Wöchentlich — RESEARCHER scannt nach neuen Whitepapers, Regulatorik-Updates, Platform-Änderungen.

Kosten: €0 (Supabase Free Tier, 500MB Vektor-Storage reicht für ~50.000 Chunks)

Schicht 2: Real-Time APIs & Tools

Was: Live-Daten die bei jedem Task-Execution frisch abgerufen werden. Marktpreise, On-Chain-Metriken, Social Trends, Platform Analytics.

APIs:

• CoinGecko (Free): Preise, Volumen, Market Cap
• Glassnode/CryptoQuant: On-Chain (Free Tier)
• Alternative.me: Fear & Greed Index
• Serper: Web Search für Breaking News
• Twitter/X API: Trending, Sentiment
• YouTube Data API: Trending Topics, Competitor Analysis
• Beehiiv API: Newsletter Metrics
• Gumroad API: Sales Data

Kosten: ~€10-15/Mo (meiste APIs haben Free Tiers, Serper €5, Rest gratis)

Schicht 3: Elite System Prompts

Was: Hardcoded Decision Frameworks der Weltbesten. Jeder Agent bekommt die Denkweise eines Top-Performers seiner Domäne einprogrammiert.

Wie: Multi-Layer System Prompts:

• Identity Layer: Wer bist du, was ist deine Rolle
• Expertise Layer: Fachspezifisches Framework
• Decision Layer: Wie triffst du Entscheidungen
• Quality Layer: Standards, Checks, Selbstkritik
• Output Layer: Format, Ton, Zielgruppe

Kosten: €0 — reine Prompt-Engineering-Arbeit. Höchster ROI aller Maßnahmen.

🔍 GraphRAG: Das Nächste Level

Standard-RAG sucht nach ähnlichen Textpassagen. GraphRAG versteht Beziehungen zwischen Konzepten — z.B. "MiCA Regulierung" → "betrifft" → "Token-Klassifizierung" → "relevant für" → "Disclaimer-Formulierung". Das ermöglicht Reasoning über verknüpftes Wissen, nicht nur Keyword-Matching.

→ Phase 2: Nach dem MVP bauen wir einen Knowledge Graph in Supabase (Nodes + Edges Tabellen). Das macht unsere Agenten 10x intelligenter bei komplexen Fragestellungen.

📈 Output-basiertes Lernen

Jeder Content-Output wird getrackt: Views, Engagement, Conversion, Revenue. Die besten Outputs werden analysiert — was hat funktioniert? Welche Daten, welcher Ton, welches Format? Diese Learnings fließen zurück in die System Prompts.

→ "Unser BTC-Report mit On-Chain-Daten hatte 3x mehr Engagement als der rein technische Report. ORACLE soll zukünftig immer On-Chain-Kontext liefern."

🏆 Competitor Benchmarking

RESEARCHER analysiert wöchentlich die Top-Performer in unserem Markt: Welche Themen covern sie? Welche Formate nutzen sie? Was können wir besser machen? Nicht kopieren — besser machen.

→ "BTC-ECHO hat ein Steuerthema coverat das 50k Views hat. Unser Guide ist tiefer — aber wir haben kein Video dazu. SCRIBE + AMPLIFIER: Video-Version produzieren."

⚠️ Wichtig: Was AI NICHT kann (und wo Christian eingreift)

Halluzination-Risiko: AI kann "kreativ" mit Fakten umgehen. Deshalb: Jeder Report durchläuft den COUNSEL Legal Check UND Christian macht stichprobenartige Factual Accuracy Reviews (15 Min/Tag).

Geschmack & Intuition: AI kann Daten analysieren, aber Christian's Markt-Intuition und Community-Gespür sind unersetzlich. Die Agenten liefern Optionen — Christian entscheidet bei strategischen Weichenstellungen.

Beziehungen: Echte Partnerships, Podcast-Auftritte, Netzwerk-Events — das ist Christian's Job. Die Agenten bereiten vor, Christian liefert den menschlichen Touch.
